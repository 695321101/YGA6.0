当前开源快照包含后端核心链路。

这个版本重点展示 YGA 如何把普通人的自然语言需求变成可检查、可评审、可持续迭代的软件生成流程。

01

Memory

管理 Session、需求记录、阶段门禁、快照和变更 pending。

02

PMC

把用户目标转为规划蓝图，并决定后续任务队列。

03

Pipeline

生成接口契约和代码，再交给 LocalGate 做确定性检查。

04

Review

对 AI 产物进行质量评审，避免只生成“看起来能跑”的项目。

前端线会把这条链路交到普通人手里。

Web 工作台和 YGA Studio 浏览器仍是后续方向，本页面展示的是产品体验目标，不代表当前仓库已经包含完整前端运行时。

现在
Python 后端链路离线 E2E 可跑，核心模块已开源。
下一步
Web 工作台需求确认、进度查看、交付入口变成可视化流程。
未来
YGA Studio 浏览器点选页面元素，用自然语言和 AI 一起改界面。
